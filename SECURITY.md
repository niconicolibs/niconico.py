# Security Policy

## Supported Versions

Security fixes are provided for the latest released version and the current `main` branch.
Older versions may receive fixes on a best-effort basis, but users should upgrade to the latest release when possible.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub Security Advisories for private disclosure when available. If private reporting is not available, contact the maintainer at <ngm1072@gmail.com> with:

- Affected version or commit
- Reproduction steps or proof of concept
- Impact and affected API surface
- Suggested fix, if known

We will acknowledge valid reports and coordinate a fix before public disclosure.
